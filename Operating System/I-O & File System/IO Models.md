完整IO流程：
1. 进程向OS发起IO调用请求
2. OS准备数据并将外部设备数据加载到内核缓冲区
3. OS将数据从内核缓冲区拷贝到用户进程缓冲区

**同步阻塞IO模型 Synchronous Blocking IO BIO**：应用程序发起I/O操作后会被阻塞，直到操作完成才返回结果。
**同步非阻塞IO模型 Synchronous Non-blocking IO NIO**：应用程序发起I/O操作后立即返回，不会被阻塞，但需要不断轮询来检查I/O操作是否完成。且第三步拷贝数据阶段是阻塞的。
**IO复用模型 / IO多路复用**：一个线程不断使用select/poll/epoll等系统调用来检查I/O操作是否完成。应用程序可以同时等待多个I/O操作，当其中任何一个I/O操作准备就绪时，应用程序会被通知。适合于需要同时处理多个I/O操作的场景，比如高并发的服务端程序。
**异步IO模型 Asynchronous IO AIO**：应用程序发起I/O操作后可以立即做其他事情，当I/O操作完成时，应用程序会得到通知（通过信号，回调函数等）。异步I/O模型由操作系统内核完成I/O操作，应用程序只需等待通知即可。适合于需要大量并发连接和高性能的场景，能够减少系统调用次数，提高系统效率。
**信号驱动IO模型**：应用程序发起I/O操作后，可以继续做其他事情，当I/O操作完成时，操作系统会向应用程序发送信号来通知其完成。适合于需要异步I/O通知的场景，可以提高系统的并发能力。


| IO多路复用 | select           | poll             | epoll                    |
| ------ | ---------------- | ---------------- | ------------------------ |
| 底层数据结构 | 数组               | 链表               | 红黑树+双链表                  |
| 获取就绪fd | 遍历fd_set         | 遍历fd_set         | 调用epoll_wait查询fd状态       |
| 复杂度    | O(N)             | O(N)             | O(1)                     |
| fd拷贝   | 每次调用从用户空间复制到内核空间 | 每次调用从用户空间复制到内核空间 | 使用内存映射mmap，无需频繁用户、内核空间拷贝 |
