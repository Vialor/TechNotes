本文简述解决以下两个问题的三种路径：
1. 代码一致性问题：自动同步/部署本地代码到服务器
2. 环境一致性问题：使用远程环境调试代码
# 工具使用
PyCharm / JetBrain Gateway：远程开发
VsCode：本地调试
# 远程开发模式

|     | 代码文件 | 解释器/调试 | 服务部署 | 应用场景                     |
| --- | ---- | ------ | ---- | ------------------------ |
| 模式A | 远程   | 远程     | 远程   | 服务器环境难复制，服务器稳定，本地工作环境不稳定 |
| 模式B | 本地   | 远程     | 远程   | 服务器环境难复制，本地工作环境稳定，服务器不稳定 |
| 模式C | 本地   | 本地     | 远程   | 服务器环境好复制，本地工作环境稳定，服务器不稳定 |

**模式 A**：JetBrain Gateway
本地只是 IDE 前端

**模式 B**：PyCharm映射工具+远程解释器（不要选自动上传项目文件到服务器）
不要选自动上传项目文件到服务器。那个只适用于想在服务器环境下跑，但是并没有真正部署的打算的情况。且此情况下不需要建立PyCharm的文件映射。
注意解释器还是运行的远程的代码，只不过会用我们本地的断点。整个调试过程我们需要借用映射工具，亲自保证代码一致性。

**模式 C**：PyCharm映射工具+本地解释器
手动维护环境一致性问题
